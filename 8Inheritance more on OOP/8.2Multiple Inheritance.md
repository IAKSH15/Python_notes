# 🧬 Python OOP – Multiple Inheritance

## ✅ What is Multiple Inheritance?

-> Multiple inheritance allows a class to inherit attributes and methods from **more than one parent class**.  
This lets a child class combine behaviors from multiple sources.

---

## 🧪 Example Code

```python
class Employee:
    company = "ITC"
    name = "Rohan"
    salary = 120000

    def show(self):
        print(f"The name of the Employee is {self.name} and the salary is {self.salary}")

class Coder:
    language = "Python"

    def printLanguage(self):
        print(f"Out of all languages here is your language {self.language}")

class Programmer(Employee, Coder):  # Multiple Inheritance
    company = "ITC Infotech"

    def showLanguage(self):
        print(f"The name is {self.name} and he is good in programming language {self.language} language")

# Object creation
a = Employee()
b = Programmer()

# Method calls
b.show()
b.printLanguage()
b.showLanguage()
```

### 📤 Output

```python
The name of the Employee is Rohan and the salary is 120000
Out of all languages here is your language Python
The name is Rohan and he is good in programming language Python language
```

---

## ✅ Explanation

|Class|Inherits From|Features Used|
|---|---|---|
|`Employee`|Base class|`name`, `salary`, `show()`|
|`Coder`|Base class|`language`, `printLanguage()`|
|`Programmer`|`Employee`, `Coder`|Inherits from both; overrides `company`, adds `showLanguage()`|

---

## 🧠 Key Concepts

-  `Programmer` has access to **both** `Employee` and `Coder` methods and attributes.
-  Python uses **Method Resolution Order (MRO)** to decide which parent to look into **first**.
-  In this case: `Programmer → Employee → Coder`

---

# 📌 Summary

|Feature|Accessed From|Output|
|---|---|---|
|`show()`|From `Employee`|Prints `name` and `salary`|
|`printLanguage()`|From `Coder`|Prints programming language|
|`showLanguage()`|In `Programmer`|Combines `name` (from `Employee`) and `language` (from `Coder`)|
