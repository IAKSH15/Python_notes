# 🧱 Python OOP – `@classmethod`

## ✅ What is a Class Method?

-  A **class method** is defined using the `@classmethod` decorator.
-  It receives the **class itself** as the first argument (named conventionally as `cls`).
-  It can **access or modify class-level attributes**, not instance-level ones.

---

## 🧪 Example Code

```python
class Employee:
    a = 1  # Class attribute

    @classmethod
    def show(cls):  # cls refers to the class, not the object
        print(f"The class attribute of a is {cls.a}")

e = Employee()
e.a = 55  # This creates a new **instance attribute** 'a' only for object `e`

e.show()
```

### 📤 Output

```python
The class attribute of a is 1
```

---

# 🧠 Explanation

|Element|Type|Value Seen by `show()`|
|---|---|---|
|`a = 1`|Class attribute|Shared by all objects|
|`e.a = 55`|Instance attribute|Overrides `a` only for `e`|
|`cls.a` inside `show()`|Class attribute|Refers to `Employee.a`, not `e.a`|

---

## 🧪 If You Used an Instance Method Instead:

```python
def show(self):
    print(self.a)
```

### 📤 Output:

```python
55
```

-> ✅ Because `self.a` accesses the instance variable.

---

# 📌 Summary Table

|Decorator|First Parameter|Accesses|Use Case|
|---|---|---|---|
|None (instance)|`self`|Instance data|Working with object-specific values|
|`@classmethod`|`cls`|Class data|Access/mod class-level variables or factory methods|
|`@staticmethod`|None|Neither|Utility functions, no object/class dependency|
