# 🧱 Python OOP – `@classmethod`, `@property`, and `@setter`

## 🧪 Example Code

```python
class Employee:
    a = 1

    @classmethod
    def show(cls):
        print(f"The class attribute of a is {cls.a}")

    @property
    def name(self):
        return f"{self.fname} {self.lname}"
    
    @name.setter
    def name(self, value):
        self.fname = value.split(" ")[0]
        self.lname = value.split(" ")[1]

e = Employee()
e.a = 55
e.show()
e.name = "Rohan Kumar"
print(e.fname, e.lname)
```

### 📤 Output

```python
The class attribute of a is 1
Rohan Kumar
```

---

## ✅ Explanation

|Decorator|Purpose|
|---|---|
|`@classmethod`|Accesses **class-level attributes** via `cls`|
|`@property`|Lets you access a **method like an attribute**|
|`@name.setter`|Allows **assignment** to the property (e.g. `e.name = ...`)|

---

## 🧠 How It Works

-  `e.a = 55`: Creates an **instance attribute**, but `cls.a` inside `show()` still refers to **class-level `a` = 1**.
-  `@property`: Lets you do `e.name` instead of `e.get_name()` to retrieve the full name.
-  `@name.setter`: Lets you do `e.name = "Rohan Kumar"` to split and set `fname` and `lname`.

---

# 📌 Summary Table

| Feature         | Code Used                                | Description                       |
| --------------- | ---------------------------------------- | --------------------------------- |
| Class method    | `@classmethod` + `cls`                   | Access class-level data           |
| Property getter | `@property` + `def name()`               | Allows `e.name` to return a value |
| Property setter | `@name.setter` + `def name(self, value)` | Allows setting `e.name = "..."`   |
