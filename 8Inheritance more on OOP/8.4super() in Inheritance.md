# 🧱 Python OOP – Constructors & `super()` in Inheritance

## ✅ What This Covers:

-  Constructor chaining with `super()`
-  Constructor overriding
-  Attribute inheritance

---

## 🧪 Example Code

```python
class Employee:
    def __init__(self):
        print("Constructor of Employee")
    a = 2

class Programmer(Employee):
    def __init__(self):
        print("Constructor of Programmer")
    b = 3

class Manager(Programmer):
    def __init__(self):
        super().__init__()  # Calls Programmer's constructor
        print("Constructor of Manager")
    c = 4

# Object creation & testing
o = Employee()
print(o.a)

o = Programmer()
print(o.a, o.b)

o = Manager()
print(o.a, o.b, o.c)
```

### 📤 Output

```python
Constructor of Employee
2
Constructor of Programmer
2 3
Constructor of Programmer
Constructor of Manager
2 3 4
```

---

## ✅ Explanation

|Class|Constructor Called|Notes|
|---|---|---|
|`Employee`|`Employee.__init__()`|Only its own constructor is called.|
|`Programmer`|`Programmer.__init__()`|Overrides `Employee`'s constructor.|
|`Manager`|`Manager.__init__()` → `super()` → `Programmer.__init__()`|Uses `super()` to call the parent class constructor explicitly.|

---

## 🧠 Key Concepts

-  `__init__()` **overrides** the parent constructor by default.
-  To call the parent’s constructor, use `super().__init__()`.
-  Attributes (`a`, `b`, `c`) are **class attributes** and are still inherited, regardless of constructor.

---

## 📌 Constructor Call Chain in `Manager`

```python
Manager()
↓ calls
Manager.__init__()
↓ calls
Programmer.__init__()  (via super())
→ Employee.__init__() is skipped unless explicitly called
```

---

## ⚠️ If you want to call all constructors in the chain:

```python
class Programmer(Employee):
    def __init__(self):
        super().__init__()
        print("Constructor of Programmer")
```

-  Then, calling `Manager()` would print:

```python
Constructor of Employee
Constructor of Programmer
Constructor of Manager
```
