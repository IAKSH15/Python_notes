# 🧱 Python Classes – Instance Attribute Overrides Class Attribute

## ✅ Concept:

-> When an **instance attribute** is assigned with the **same name** as a class attribute, the **instance-level value overrides** the class-level one **only for that object**.

---

## 🧪 Example Code

```python
class Employee:
    language = "Python"
    salary = 120000

rohan = Employee()
rohan.language = "Java"  # Overrides class attribute for this instance

print(rohan.language, rohan.salary)
```

### 📤 Output

```python
Java 120000
```

---

## 🧠 Explanation

|Attribute|Source|Behavior|
|---|---|---|
|`language`|Instance attribute|`"Java"` (overrides class value)|
|`salary`|Class attribute|Inherited from class|

-  `rohan.language = "Java"` creates a **new instance attribute**.
-  `language` at class level still remains `"Python"` for other instances.

---

## 🔍 Proof – Compare with another instance

```python
mohan = Employee()
print(mohan.language)  # Still shows "Python"
```

### 📤 Output

```python
Python
```

---

# 📌 Key Rule:

-> **Instance attributes take precedence** over class attributes with the same name **at the object level**.
