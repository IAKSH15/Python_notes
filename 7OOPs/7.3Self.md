# ЁЯз▒ Python Classes тАУ Methods, Static Methods, and Attribute Behavior

## тЬЕ Objective

Understand:

-  Regular methods (`self`)
-  Static methods (`@staticmethod`)
-  How instance attributes override class attributes

---

## ЁЯзк Example Code

```python
class Employee:
    language = "Python"
    salary = 120000

    def getinfo(self):
        print(f"The language is {self.language}.\nThe salary is {self.salary}")

    @staticmethod
    def greet():
        print("Good Morning")

rohan = Employee()

rohan.getinfo()         # Calls instance method
rohan.language = "Java" # Overrides class attribute for this object
rohan.greet()           # Calls static method
rohan.getinfo()         # Shows updated language
```

### ЁЯУд Sample Output

```python
The language is Python.
The salary is 120000
Good Morning
The language is Java.
The salary is 120000
```

---

## тЬЕ Explanation

|Feature|Description|
|---|---|
|`getinfo(self)`|An **instance method**, requires `self`, and accesses instance/class data.|
|`@staticmethod`|A method that **does not take `self`**, can be called without any instance.|
|`rohan.language`|Overrides the class attribute only for `rohan`.|

---

## ЁЯза How Methods Behave

-  `rohan.getinfo()` is the same as:

```python
Employee.getinfo(rohan)
```

-  `self` refers to the **calling object** (here, `rohan`).
-  Static methods **do not need** `self` and are used for **utility functions**.
