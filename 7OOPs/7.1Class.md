# 🧱 Python OOP – Class vs Instance Attributes

## ✅ Objective:

-> Understand the difference between **class attributes** and **instance attributes** using a simple `Employee` class.

---
## 🧪 Example Code

```python
class Employee:
    language = "Python"    # Class attribute
    salary = 120000        # Class attribute

rohan = Employee()
rohan.name = "Rohan"       # Instance attribute
print(rohan.name, rohan.language, rohan.salary)

mohan = Employee()
mohan.name = "Mohan"       # Another instance
print(mohan.name, mohan.salary, mohan.language)
```

### 📤 Output

```python
Rohan Python 120000
Mohan 120000 Python
```

---

# 🧠 Key Concepts

|Attribute Type|Defined Where?|Shared Between Instances?|Example|
|---|---|---|---|
|**Class Attribute**|Inside class (directly)|✅ Yes|`salary`, `language`|
|**Instance Attribute**|Assigned to specific object|❌ No (unique per object)|`name` (like `rohan.name`)|

---

# 📌 Summary

-  `language` and `salary` are **class attributes**, shared by **all objects** of the class.
-  `name` is an **instance attribute**, unique to each object (`rohan`, `mohan`).

---

## 💡 Pro Tip:

-> If you assign a **class attribute** via an instance (e.g., `rohan.language = "Java"`), it will override it **only for that instance**, creating a new **instance attribute** with the same name.
