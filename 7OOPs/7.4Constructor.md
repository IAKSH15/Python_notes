# 🧱 Python OOP – `__init__()` Constructor and Object Initialization

## ✅ Objective

-> Use the `__init__()` method to initialize attributes automatically when creating an object of a class.

---

## 🧪 Example Code

```python
class Employee:
    language = "Python"
    salary = 120000

    def __init__(self, name, salary, language):
        self.name = name
        self.salary = salary
        self.language = language
        print("I am creating an object")

    def getinfo(self):
        print(f"The language is {self.language}.\nThe salary is {self.salary}")

    @staticmethod
    def greet():
        print("Good Morning")

rohan = Employee("Rohan", 50000, "Java")
print(rohan.name, rohan.salary, rohan.language)

# parth = Employee()  # ❌ This will raise an error: missing arguments
```

### 📤 Output

```python
I am creating an object
Rohan 50000 Java
```

---

## ✅ Explanation

|Concept|Detail|
|---|---|
|`__init__()`|A **constructor method** automatically called when an object is created|
|`self`|Refers to the **current object** being created|
|`rohan = Employee(...)`|Initializes `name`, `salary`, and `language` for the `rohan` object|
|`print(...)`|Confirms object was created and initialized|

---

### 🧠 What Happens Internally?

-> When you call:

```python
rohan = Employee("Rohan", 50000, "Java")
```

-> Python does:

```python
1. Allocate memory for `rohan`
2. Call `__init__` with the arguments
3. Set: 
   rohan.name = "Rohan"
   rohan.salary = 50000
   rohan.language = "Java"
```

---

## ❌ Error If Missing Arguments:

```python
parth = Employee()
```

### 📤 Output:

```python
TypeError: __init__() missing 3 required positional arguments: 'name', 'salary', and 'language'
```

# 🧠 Summary Table

|Element|Type|Purpose|
|---|---|---|
|`__init__()`|Constructor|Initializes object attributes|
|`self`|Reference|Refers to current instance (`rohan`)|
|`@staticmethod`|Utility method|Doesn't depend on instance attributes|
